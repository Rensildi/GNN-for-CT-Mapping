# Experiment 1 — Overview

Companion write-up to [`architecture_exp1.png`](architecture_exp1.png). Explains the dataset, labels, preprocessing, hypotheses, architectural rationale, and expected results for the GCN-vs-MLP baseline experiment.

![Experiment 1 architecture](architecture_exp1.png)

## 1. Dataset — LIDC-IDRI

The Lung Image Database Consortium and Image Database Resource Initiative (LIDC-IDRI) is the canonical open dataset for lung nodule analysis. Distributed by The Cancer Imaging Archive.

- **Patient cohort:** 1,010 patients, 1,308 CT studies, 244,527 total DICOM slices (~133 GB).
- **Annotation protocol:** two phases per scan, performed by four thoracic radiologists independently.
  - *Blinded phase:* each radiologist marks "nodule ≥ 3 mm," "nodule < 3 mm," or "non-nodule ≥ 3 mm."
  - *Unblinded phase:* they see peers' marks and can revise — but **no forced consensus**. The same anatomical nodule may be rated by 1–4 readers, and the number of raters per nodule is a real data-distribution feature, not noise.
- **Ground truth:** primarily radiologist consensus (subjective). A subset of ~157 nodules has biopsy- or surgery-confirmed diagnoses; these are the closest thing to pathology ground truth and are a secondary eval axis in later experiments.

### What's in this repo

- `GNN_for_CT_Mapping/data/annotations/tcia-lidc-xml/` — 1,318 LIDC XML annotation files (gitignored, local only).
- `GNN_for_CT_Mapping/data/annotations/tcia-diagnosis-data-2012-04-20.xls` — pathology-confirmed diagnoses sheet (used later).
- `GNN_for_CT_Mapping/data/nodules.parquet` — generated by `scripts/preprocess.py`, **gitignored** (regenerate locally).
- `GNN_for_CT_Mapping/data/splits/fold_{0..4}.json` — 5-fold CV assignments, **committed** (reproducibility).

## 2. Labels

**Primary label (binary):** malignant vs. benign.

Derived from the mean of up to four radiologists' 1–5 malignancy ratings:

```
mean ≤ 2  →  benign (0)
mean ≥ 4  →  malignant (1)
otherwise →  excluded (ambiguous, ~score 3)
```

The ≥ 4 / ≤ 2 threshold is standard in the LIDC literature and keeps labels on the confident end of the scale. Excluded middle-score nodules are filtered *before* training — they never appear in `nodules.parquet`.

**Not labels — used as features:** the eight LIDC-IDRI *characteristic* ratings (subtlety, internal structure, calcification, sphericity, margin, lobulation, spiculation, texture). These are 1-based integer Likert-style ratings on per-reader scales (mostly 1–5, with internal structure 1–4 and calcification 1–6). In our pipeline they feed the clinical branch of Stage 1 and are deliberately tracked separately from malignancy to prevent accidental label leakage.

**Secondary labels (future eval axis):** the ~157 pathology-confirmed diagnoses from `tcia-diagnosis-data-2012-04-20.xls`. Not used for Experiment 1's headline metric but reserved for pathology-subset reporting in Experiment 3.

## 3. Data preprocessing

All preprocessing is deterministic at `seed=42` and lives under `experiments/harrison/scripts/`. Three stages:

### 3a. XML parsing (no pylidc)

`scripts/parse_lidc_xml.py` — pure-stdlib `xml.etree.ElementTree` parser for the LidcReadMessage schema. Produces, per XML file:

- Scan-level identifiers (SeriesInstanceUID, StudyInstanceUID).
- A flat list of per-reader nodule records, each containing:
  - The reader's 0-indexed position (0 .. 3).
  - The 8 descriptive attribute ratings (0-indexed so they index embeddings directly).
  - Malignancy (**kept in its own field**, not in the attribute dict — prevents the 8-attribute feature path from accidentally including the label).
  - The per-slice ROI contours (`imageZposition` in mm + per-point `(xi, yi)` pixel indices on that slice).

### 3b. DICOM loading + 3D patch extraction

`scripts/dicom_loader.py` uses `pydicom` to:

1. Load every DICOM file in a series directory.
2. Sort slices by physical Z (`ImagePositionPatient[2]`) — more robust than InstanceNumber, which some vendors reverse.
3. Apply per-slice `RescaleSlope` / `RescaleIntercept` to convert raw pixel values to **Hounsfield Units**.
4. Stack into a `(Z, Y, X)` volume, capturing voxel spacing `(z, y, x)` in mm and the physical origin.
5. Convert physical (x, y, z) centroids to voxel indices and extract a **48³-voxel patch** centered on each nodule (padding out-of-bounds regions with −1000 HU, which is air — the physically meaningful extension).

Feature-extraction-time preprocessing (applied inside `scripts/extract_features.py`):

- **HU clip** to `[-1000, 400]` (air to soft-tissue window).
- **Normalize** to `[0, 1]` via `(hu + 1000) / 1400`.
- **Resample** to isotropic 1 mm³ via linear interpolation so the Med3D receptive field is geometrically comparable across scanners with different slice thicknesses.

### 3c. Cross-reader nodule clustering

`scripts/preprocess.py` merges per-reader annotations into unified nodules:

1. **Compute centroid in mm** per per-reader annotation from the edge-map contours (mean of the per-slice pixel means, converted to physical space).
2. **Greedy centroid-distance clustering** with an **8 mm threshold** — a simpler heuristic than pylidc's IoU-based clustering but sufficient at LIDC's scale (~0.5% of our output has > 4 merged annotations, indicating a mild over-merge rate worth watching).
3. **Aggregate** per cluster: mean malignancy → binarize, per-attribute mean, majority patient ID via `metadata.csv`.
4. **Label** per cluster via the binarization rule above; drop ambiguous clusters.

### 3d. Cross-validation splits

`scripts/build_splits.py` — **patient-level** 5-fold CV via `sklearn.model_selection.StratifiedGroupKFold`:

- `groups = patient_id` — no patient's nodules can straddle train/val.
- Stratification uses each patient's *majority* label across their nodules; ties break to malignant so the minority class stays balanced.
- Output: `data/splits/fold_{0..4}.json` with per-fold train/val patient lists and class counts.

Reproducibility guarantees:

- `seed = 42` everywhere (torch, numpy, random, StratifiedGroupKFold).
- `torch.backends.cudnn.deterministic = True`.

**Inductive graph construction** (Stage 2, applied per fold):

- KNN edges are fit on **train-fold nodules only**.
- Val nodes are inserted as new graph nodes whose edges point only at their k nearest training neighbors — **no val-to-val edges, no retroactive train-node edges to unseen val nodes**. This is the correct inductive setup for a cohort-wide graph under patient-level CV; an accidental transductive setup is the #1 way to inflate results in this project.

## 4. Hypotheses

**H1 (primary, one-sided):** the 2-layer GCN achieves a higher mean macro-AUC across the 5 validation folds than the parameter-matched MLP baseline by **≥ 0.01**, with a paired Wilcoxon signed-rank **p < 0.05**.

**H0 (null):** no significant difference — graph structure doesn't meaningfully help on LIDC when both models see the same fused multi-modal features.

**Secondary expectations** (not gates, just things we'll check):

- MLP AUC is in the vicinity of 0.88–0.92 — that's the range established by published LIDC classifiers with roughly this feature set.
- GCN improvement, if present, is small (≤ 0.03 AUC). Large gains would be surprising given that the fused features are already strongly discriminative.
- AUPRC should move in the same direction as AUC. If they diverge sharply, the graph is probably helping (or hurting) at an unusual operating point and the reliability diagram will say why.

## 5. Why the architecture fits the task

### 5a. Multi-modal fusion captures complementary signal

Each nodule has three kinds of information that don't reduce to each other:

- **Imaging features** — sub-mm morphology that radiologists compress into their categorical attributes but don't fully verbalize. Med3D's pretrained 3D convolutions give us these for free.
- **Clinical attributes** — the radiologist's expert compression of features they *can* verbalize. Including them is a hedge: even if the image branch underperforms on an ambiguous nodule, the attributes carry actionable prior information. (Caveat — see §6 below on label leakage.)
- **Spatial location** — upper-lobe nodules have elevated malignancy rates at population level. Sinusoidal positional encoding captures this without privileging any particular axis.

Concatenating all three branches, LayerNorm'ing (because they have very different natural scales), and projecting to a common `node_feature_dim = 256` gives the Stage 2 head a consistent input shape regardless of which modality drives the signal in a given nodule.

### 5b. Frozen Med3D matches the data regime

We have ~1,128 labeled nodules after binarization. Training a 3D CNN from scratch at that scale is a recipe for overfitting. Med3D (Chen et al. 2019) is pretrained on 23 medical segmentation datasets — it's already seen millions of 3D medical patches — so using it as a frozen feature extractor lets us absorb 3D priors without ever updating its weights on our tiny label set. 48³-voxel patches at isotropic 1 mm³ match the receptive-field scale Med3D was pretrained for.

Keeping the backbone frozen also makes Stage 1 *cheap enough* to run once and cache to parquet: the whole training loop (GCN + MLP across 5 folds) then becomes small-tensor arithmetic that comfortably fits on a 12 GB 3060.

### 5c. Inductive bias of the GCN

The hypothesis motivating the graph head is simple: **similar nodules have correlated labels**. If two nodules share morphology, attributes, and anatomical location, their malignancy labels should be correlated — and an ambiguous nodule can borrow confidence from clearly-labeled lookalikes.

A 2-layer GCN operationalizes this:

- *Layer 1* aggregates immediate neighbors into each node's representation.
- *Layer 2* extends to 2-hop neighbors (nodules similar to nodules similar to you).
- Self-loops + symmetric normalization preserve each node's own features while weighting neighbor contributions.

Two layers is the standard choice for node classification on small graphs — deeper stacks over-smooth (all node representations collapse toward the graph mean).

### 5d. Parameter-matched MLP isolates the graph effect

If we just reported "GCN beats a small logistic regression," capacity would explain the win. The MLP baseline has the **same layer widths, same dropout, same activation** as the GCN — the only difference is that its first two layers are `Linear(N, M)` instead of `GCNConv(N, M)`. Any AUC delta therefore isolates the contribution of message passing over the KNN graph, not raw model capacity.

### 5e. Inductive eval eliminates the usual cohort-graph pitfall

Patient-level splits + a cohort-wide graph is easy to get wrong. If val nodes participated in the graph at training time, the GCN effectively trains on val labels via message-passing gradients — a subtle transductive leak. Our setup forbids this: edges are fit on train-only features, and val nodes are inserted at eval time with edges only into train. We verified no train/val patient leakage at the fold level.

## 6. Expected results

### Headline outcomes

Prior art constrains the space:

- **MLP baseline** should land around **AUC 0.88–0.92**. That's the range published LIDC nodule classifiers hit with roughly our feature set — image patches + radiologist attributes.
- **Ma et al. 2023** — the closest prior work using a GCN on LIDC — reports **AUC 0.9629**. Their setup is different from ours (GCN for cross-CNN feature fusion, not for node classification on a similarity graph), so their number is a useful reference but not a direct target.

Three plausible outcomes, in rough order of prior probability:

1. **Narrow GCN win (AUC +0.005 to +0.02, inconsistent significance).** Most likely. The graph adds a small regularization signal on ambiguous cases, but LIDC features are already separable enough that the ceiling isn't far above the MLP.
2. **Null result (AUC within ±0.005, Wilcoxon non-significant).** Also plausible — the KNN graph is built on the same feature space the GCN then smooths over, so the GCN is partly redundant with the Stage 1 projection. If this happens, Experiment 2's graph-construction ablation becomes the interesting knob.
3. **GCN loses slightly.** Would indicate the KNN graph is noisy enough (wrong neighbors) that message passing hurts. Watch for this via the degree distribution and edge-length stats in TensorBoard.

### What would change the interpretation

- **The 8-attribute branch is a potential label-leak risk.** Malignancy scores and the 8 descriptive attributes come from the same radiologists at the same reading session, so they're correlated by construction. Experiment 3's image-only feature-modality ablation is the clean test of whether the image branch is learning something the attributes don't already encode. Any Experiment 1 result that looks "too good" should be flagged until Experiment 3 confirms it.
- **Pathology-confirmed subset eval.** Our headline metric uses radiologist-consensus labels — opinion, not ground truth. The ~157 pathology-confirmed nodules (biopsy / surgery / follow-up) are the real gold standard. Performance on that subset may differ meaningfully from the full-cohort number.

- `outputs/predictions/exp1_{gcn,mlp}_fold{0..4}.parquet` — per-nodule probability + label, ready for significance testing.
- `outputs/predictions/exp1_summary.parquet` — per-fold AUC/AUPRC/Brier.
